Function for chess openings

Standard case:

        We have a computational resource and space for data;

        We use standard chess software with a ready file of chess openings;

        No problems, good chess.

Other case:

        We don’t have enough resource, network is limited, there is no file access;

        There is no way to implement any database;

        Can’t calculate good chess.

Of course, such situation sounds bit impossible. No at all. Some examples:

Simple javascript in browser (like offline tutorials);

        Scripts for 3D (no databases available), scripts are slow;

        Language hasn’t binary operators;

        Or just want to play first chess moves correctly - in no matter what kind of environment.

.

How to solve:

        Declare a variable and assign data of well known openings;

        Create function that generates best chess variants, if possible for given request;

        Size of code should be minimal and reasonable - some Kbytes.

Is it possible at all?

There are 2 ways to hold/generate chess variants.

        1. As a tree;

        2. As a perimeter of most used variants.

Tree structuring sounds natural, but... All the nodes and separators take space too. And quite a lot, if no binary operators available.

        

So,  here is the other algorithm:

We define path-perimeter of most used variants. The notation should be short, no long sidewalks, line should be straight. Then data are like short steps to the right/left - like values of points.

Of course, nothing is perfect.

Anyway, it works even for long mainstream variants.
